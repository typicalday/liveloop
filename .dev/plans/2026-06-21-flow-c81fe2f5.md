# Heartbeat Leases — Build Plan

## Goal
Add a real liveness signal (heartbeat_at) to the engine so a long-running producer that periodically reports in is never falsely reaped; unify the three staleness predicates; add per-loop TTL override; surface per-step attempts in status debts.

**New private helper — isClaimFresh(task: TaskData, now: number, ttl: number): boolean**
This is the unified liveness predicate:
```typescript
private isClaimFresh(task: TaskData, now: number, ttl: number): boolean {
  const anchor = task.heartbeatAt !== undefined && task.heartbeatAt > (task.claimedAt ?? 0)
    ? task.heartbeatAt
    : (task.claimedAt ?? 0);
  return now - anchor <= ttl;
}
```
Formula: `now - max(claimedAt, heartbeatAt ?? claimedAt) <= ttl`. Returns true if fresh (not stale).

**New public method — Engine.heartbeat(workflow, run, now?)**
Add between `close()` and `reap()`:
```typescript
heartbeat(workflow: string, run: string, now?: number): void {
  const ts = now ?? nowMs();
  this.store.tx(() => {
    // openRun enforces: exists, not closed, task.run === run
    const r = this.openRun(workflow, run);
    this.store.touchHeartbeat(workflow, r.loop, r.key ?? '', ts);
  });
}
```
`openRun` throws if the run no longer holds its lease — zombie prevention is already enforced.

## Edge Cases

1. **heartbeatAt older than claimedAt** (clock regression): `max(claimedAt, heartbeatAt ?? claimedAt)` uses claimedAt as floor. Safe.
2. **heartbeat on reaped run**: `openRun` checks `task.run === run`. After reap, `putTask` writes `status: 'idle'` with no `run` field, so `task.run` is undefined, not equal to the old run ID. Throws. Correct.
3. **heartbeat on superseded run**: task.run now points to newer run. Old run ID doesn't match → throws. Correct.
4. **touchHeartbeat with no matching task row**: UPDATE is a no-op (0 rows). Since `openRun` already verified the task exists, this is unreachable in normal flow. Safe.
5. **reap with def threading**: `reap` is always called from within `this.store.tx(...)` in `tick`, where `def` is already resolved outside the tx. Pass it in to avoid a second `defFor` call inside the tx.
6. **isInFlight def threading**: `computeTimeFacts` signature already includes `def`. No extra lookup needed.
7. **heartbeat EngineEvent**: no new event type. Heartbeat changes no artifact state; firing `settled` would be misleading. Consistent with how `close()` fires `closed` but not `settled`.
8. **SCHEMA_VERSION**: bump from `'3'` to `'4'` in store.ts to reflect the schema addition.
9. **attempts in status debts for map loops**: the key recovery logic (`parseElement` → `elementPath`) is the same logic already used for `failedRuns` enrichment. Correct.
10. **attempts=0 suppression**: only set `d.attempts` when `task.attempts > 0`. No clutter for the common case.

## Tests (new file test/heartbeat.test.ts)

All tests use the real Engine over `:memory:` store (same pattern as engine.test.ts). Import `def`, `loop`, `input` from `./helpers.ts`.

### Test 1 — beating run survives past the global TTL
- Engine with `reapTtlMs: 1000`; delivery def (planner → plan).
- Tick at t=0 → claim R1 for planner.
- Heartbeat at t=500; heartbeat at t=1200 (500ms after previous beat, within TTL).
- Tick at t=2000 (total runtime 2000ms >> global TTL 1000ms, but last beat 800ms ago < TTL).
- Assert `reaped === 0`.

### Test 2 — beat from reaped run throws
- Engine with `reapTtlMs: 100`.
- Tick at t=0 → get run ID R1.
- Tick at t=200 (no beat, past TTL) → reaped=1.
- `engine.heartbeat(wf, R1, 201)` → assert throws matching /no longer holds its lease|reaped or superseded/.

### Test 3 — beat from superseded run throws
- Engine with `reapTtlMs: 1000`.
- Tick at t=0 → R1 claimed.
- close R1 with outcome 'ok'.
- Tick at t=100 → R2 claimed (new run for same loop).
- `engine.heartbeat(wf, R1, 200)` → throws (task.run is now R2).

### Test 4 — non-beating run past its window is reaped
- Engine with `reapTtlMs: 100`.
- Tick at t=0 → R1 claimed. No heartbeats.
- Tick at t=200 → assert reaped=1.

### Test 5 — per-loop TTL override shorter than engine default
- Engine with `reapTtlMs: 2000`; planner loop has `reapTtlMs: 500`.
- Tick at t=0 → R1 claimed. No heartbeats.
- Tick at t=800 (past loop TTL 500ms, before engine TTL 2000ms) → assert reaped=1.

### Test 6 — per-loop TTL override longer than engine default
- Engine with `reapTtlMs: 500`; planner loop has `reapTtlMs: 2000`.
- Tick at t=0 → R1 claimed. No heartbeats.
- Tick at t=800 (past engine TTL 500ms, before loop TTL 2000ms) → assert reaped=0.

### Test 7 — status() exposes attempts and it increments after each reap
- Engine with `reapTtlMs: 100`, delivery def.
- Tick at t=0 → R1 for planner.
- Tick at t=200 → reaped=1.
- `engine.status(wf).debts` → find debt for 'plan' → assert `attempts === 1`.
- Tick at t=300 → R2 claimed. Tick at t=500 → reaped=1 again.
- Assert `attempts === 2`.
